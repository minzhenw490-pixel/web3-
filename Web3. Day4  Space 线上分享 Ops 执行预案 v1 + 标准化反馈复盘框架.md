# Web3 Space 线上分享 Ops 执行预案 v1 + 标准化反馈复盘框架
## 一、Space 全流程执行预案 v1
### （一）活动前期筹备（活动前24h~72h）
#### 1. 需求确认 & 基础信息归档
统一归档信息：
1. Space 主题、核心叙事（生态分享/项目AMA/开发者工坊）
2. 开播时间、时区、预计时长、主持人/嘉宾名单
3. 核心KPI：新增社群人数、Twitter曝光、builder报名、提问互动量
4. 渠道投放清单：TG社群、Twitter预热帖、合作项目转发、实习学员触达
5. 配套物料：海报、预告文案、嘉宾简介、提问收集表单、话术脚本

#### 2. 嘉宾对接分工
1. T-48h：确认嘉宾到场时间、发言时长、发言提纲、Q&A环节安排
2. T-24h：发送Space测试链接，校验麦克风、网络、账号开播权限
3. T-12h：二次开播提醒，同步完整流程表、对接主主持人

#### 3. 预热投放执行时间线
- T-3天：首条预热推文，放出嘉宾阵容+活动福利，全合作社群同步
- T-1天：二次预热，上线提问收集表单，整理社区前置问题
- T-2h：开播倒计时推文，社群轮播提醒直播间入口
- T-30min：最后一轮催场推送，引导听众提前入场

#### 4. 定岗分工清单
| 岗位 | 核心职责 | 兜底备用方案 |
| ---- | ---- | ---- |
| 主主持人 | 开场串场、嘉宾引导、全场节奏把控、收尾总结 | 备用小号，主号掉线立刻切换 |
| 社群运营 | TG实时答疑、同步直播重点、收集现场提问、秩序维护 | 多账号分群同时值守 |
| 内容记录员 | 全程录音、实时文字纪要、截图关键发言、提取金句 | Notion在线共享文档实时更新 |
| 技术兜底 | 处理掉线、麦克风故障、网络崩溃，切换备用语音渠道 | 备用TG语音频道兜底 |
| 社媒同步员 | 实时分段发推，输出分享核心观点，外部引流 | 提前预制分段短句文案 |

#### 5. 前置风险应急方案
1. 嘉宾迟到/临时缺席：启用储备社区问题，延长问答环节过渡
2. Space服务器卡顿崩溃：立刻切换TG语音频道，社群同步备用入口
3. 水军恶意刷屏攻击：运营快速禁言拉黑，主持人转移话题弱化处理
4. 直播间冷场无提问：调用表单预收集问题，运营小号主动抛问带动氛围
5. 分享超时：开播前10分钟提醒嘉宾，压缩自由讨论快速收尾

### （二）活动中实时执行流程
1. 开播前10min：全员就位，设备测试，确认嘉宾在线
2. 0~5min 标准开场：自我介绍→本期主题介绍→嘉宾逐一介绍→说明互动规则与福利
3. 嘉宾主体分享阶段：同步实时纪要、社群同步重点、社媒实时播报引流
4. Q&A互动：筛选有效问题有序交付嘉宾，过滤无关负面提问
5. 收尾5min：内容总结、下期预告、福利领取指引、引导留存社群
6. 正式结束：社群发布收尾通知，告知纪要/录音产出时间

### （三）活动收尾执行（结束0~24h）
1. 结束1h内：导出录音、整理实时纪要、截取高光截图、社群发放福利
2. 结束6h内：统计完整活动数据，填充复盘表格
3. 结束24h内：发布完整纪要推文，同步全渠道知识库、合作社群，私信活跃builder维护关系
