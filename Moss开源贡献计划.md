# Open Source Contribution Plan | Moss开源贡献计划
## 1. Builder 身份
Operation Builder 运营贡献者
暂无正式Web3项目运营履历，擅长结构化信息整理、文档编写、社群答疑，偏向零代码内容向贡献，依托笔记梳理能力产出新手学习物料，辅助降低项目参与门槛。

## 2. 选定贡献方向
文档内容共建：完善Moss新手配套学习资料，完成FAQ合集迭代、补充入门科普内容，持续收集社群高频问题更新文档。

## 3. 选择该方向的原因
1. 与个人能力匹配：我已完成完整Moss新手FAQ文档，擅长文字梳理，无需深厚代码能力，可快速落地持续产出；
2. 项目存在刚需：当前Moss缺少统一新手答疑文档，新人常卡在环境配置、PR规范、AI开发等问题，FAQ能直接解决社区痛点；
3. 成长适配：运营是我的发展方向，持续整理项目文档、对接社群问题，能同步积累Web3社区运营实战经验；
4. 生态价值：标准化学习物料可吸引更多新人参与开源，减轻核心开发团队答疑压力，扩大Moss社区共建规模。

## 4. 本周完整贡献计划
### 本周核心目标
基于已完成的FAQ文档进行优化迭代，补充配套科普短文，同步收集社群问题完善文档，产出可对外公开的完整学习素材。
- Day1：复盘现有FAQ文档，对照MOS项目会议纪要补充遗漏知识点，修正模糊、晦涩的解答；
- Day2：本地完整运行Moss模拟Demo，补充实操场景相关问答，完善环境报错类FAQ；
- Day3：撰写一篇Moss科普介绍短文，搭配FAQ作为配套学习资料；
- Day4：进入官方社群收集用户真实提问，新增5-10条高频问题至FAQ文档；
- Day5：统一排版两份文档，补充操作说明文字，优化阅读逻辑；
- Day6：将文档上传GitHub并生成HackMD公开链接，在社群发布收集反馈；
- Day7：汇总全部产出，整理成果清单，完成本周贡献复盘。

## 5. 预计产出
1. 优化完整版 Markdown《Moss新手FAQ合集》（基于已完成初稿迭代）
2. Markdown格式Moss项目科普介绍文章1篇
3. 社群用户问题收集清单1份
4. 文档公开访问链接（HackMD/Notion）
5. 配套社群推广简短科普文案,用于社区分发
