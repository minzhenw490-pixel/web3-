# Week1 Build Log | Web3 Career Build 第一周阶段总结
## 基础信息
- 学习周期：Week 1
- 分轨选择：Ops 方向
- 文档用途：阶段实践记录 & Week2 学习规划前置材料

---

## 一、本周完成链上实践记录
1. 新建加密钱包，完成助记词、私钥备份，掌握钱包基础安全操作
2. 前往测试网水龙头申领测试代币，持有链上基础资产
3. 使用 Remix IDE 编写简易Solidity智能合约，完成基础业务逻辑开发
4. 将合约部署至测试网，执行合约调用、链上交互完整流程

## 二、实践中遇到的问题
1. 测试网节点连接不稳定，钱包频繁无法联网，交互页面加载失败
2. 平台、水龙头、IDE工具大量英文专业界面，专业术语理解门槛高
3. AI生成的Solidity合约代码存在语法、逻辑错误，无法直接编译部署
4. Gas参数、RPC节点配置不熟悉，多次出现交易打包超时、部署失败

## 三、AI辅助学习记录
1. 页面英文翻译、分步操作指引：复制工具页面内容，由AI拆解操作步骤
2. 智能合约基础模板生成：通过提示词产出初始积分合约代码
3. 报错故障排查：将编译报错、网络异常信息发给AI定位问题根源
4. Web3专业词汇释义，降低外文工具阅读难度

## 四、必须人工自主判断、手动修正的环节
1. AI输出合约代码存在漏洞，需逐行人工校验语法与业务逻辑，手动修复报错
2. Gas费、部署参数需根据当前测试网状态人工调整，AI通用参数无法直接复用
3. 私钥、助记词安全存储操作，只能自行把控，无法依靠AI完成安全管控
4. 本地网络连接故障，需手动切换RPC、更换浏览器测试，AI无法远程解决本地环境问题

## 五、对 Monad 与 Web3 的认知变化
1. Monad生态：从完全陌生到完整走完测试网全流程，熟悉水龙头、合约部署、链上交互整套生态工具使用逻辑
2. Web3行业认知：打破片面认知，意识到Web3是需要持续终身学习的领域，技术、运维、研究知识迭代速度快
3. 实操认知：链上开发不存在一次成功，网络、代码、节点各类故障频发，问题排查是核心基础能力

## 六、选择Ops方向的理由
1. 兴趣层面：对底层Tech合约开发兴趣较低，不愿长期深耕底层编码
2. 能力适配：实操中更擅长网络调试、节点连接、故障排查、环境部署类运维工作
3. 长期规划：Ops赛道聚焦测试网运维、节点管理、链上排错，与本周实操遇到的网络、部署类问题高度匹配，适合长期深耕
4. 学习偏好：更喜欢环境搭建、流程优化类实操任务，而非长期编写底层合约代码

## 七、本周核心学习收获（3点）
1. 完整跑通「钱包创建→领取测试币→合约编写→部署→链上交互」全链路，建立Web3基础实操体系认知
2. 掌握借助AI快速获取指引的方法，同时学会人工校验、修复AI错误代码的基础排错能力
3. 清晰区分 Tech / Ops / Research 三大赛道定位，明确自身适配Ops方向，锁定Week2深耕路线

## 八、需要助教/同伴协助解决的问题
1. 测试网RPC节点稳定优化方案，彻底解决钱包无法联网交互的问题
2. Solidity合约高频报错快速排查技巧，简化代码纠错流程
3. Ops赛道Week2针对性学习路线、配套实操项目推荐

## 九、Week2 计划深耕方向
聚焦Web3 Ops相关实操：测试网节点运维、RPC配置管理、链上故障排查、合约部署运维流程优化，减少底层合约编码学习，重点打磨链上运维实操能力
